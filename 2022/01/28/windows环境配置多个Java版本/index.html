windows环境配置多个Java版本

发表于 2022-01-28 分类于 环境配置 ， java
本文字数： 1.1k 阅读时长 ≈ 1 分钟

0x01下载安装包

​ 本次安装已Java8和Java17为例。Java8因为只提供exe的安装程序，所以只能下周再安装包。而Java17就下载压缩包就好。

0x02安装Java8

安装jdk

​ 运行安装包进行安装即可。

安装jre

​ 选择安装到与jdk同一目录，找起来比较方便。

配置环境变量

​ 一共需要配置三项内容：JAVA_HOME、CLASSPATH和PATH。在系统变量中进行修改。

JAVA_HOME：

​ 变量名：JAVA_HOME

​ 变量值：选择刚才安装jdk的目录，默认为：C:\Program Files\Java\jdk1.8.0_131

CLASSPATH：

​ 变量名：CLASSPATH

​ 变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;

PATH：

​ 双击进入编辑界面，然后点击右侧新建按钮，新建两条：

​ %JAVA_HOME%\bin

​ %JAVA_HOME%\jre\bin

0x03安装Java17

解压文件

​ 先解压Java17的压缩包，建议把解压文件放到与Java8同一目录下。

文件配置

​ Java17已经没有jre这个文件夹了。如果需要，我们要输入一下命令。

​ bin\jlink.exe --module-path jmods --add-modules java.desktop --output jre

配置Java17环境

​ 由于CLASSPATH和PATH刚才都已经配置过，现在只需要配置JAVA_HOME就可以了。

系统变量中新建变量JAVA17_HOME

​ 变量名：JAVA17_HOME

​ 变量值：刚才防止Java17解压文件的目录，C:\Program Files\Java\jdk-17.0.2

配置JAVA_HOME

​ 刚才我们在java8配置的JAVA_HOME，改成JAVA8_HOME

​ 最后我们再重新创建一个变量JAVA_HOME

​ 变量名：JAVA_HOME

​ 变量值：%JAVAxx_HOME%

​ 当我们想要使用哪一个版本的Java，就把JAVAxx_HOME的xx改为需要的版本就可以了。

修改javapath

​ 到上面一步还并没有完全实现，还需要进行最后一步修改。当我们把JAVA_HOME改成17后，进cmd查看Java版本还是Java8的。

​ 这是因为path中的第一行javapath的优先级高于后面我们设置的 JAVA_HOME的内容。而javapath是指向Java8的版本。所以我们要将我们添加的两行内容放到javapath那一行的上面就可以了。
